连接查询是将两个或者两个以上的表按照某个条件连接起来，从中选取需要的数据
##内连接查询
查询emploee和department
```
mysql> SELECT* FROM emploee;
+------+--------+------+------+----------+------+
| num  | name   | age  | sex  | homeaddr | d_id |

|    1 | 张三   |   26 | 男   | 北京     | 1001 |
|    2 | 李思   |   24 | 女   | 北京     | 1001 |
|    3 | 王五   |   25 | 男   | 长沙     | 1002 |
|    4 | Aric   |   15 | 男   | England  | 1002 |
|    1 | 天天   |   22 | 男   | 南京     | 1001 |
+------+--------+------+------+----------+------+
5 rows in set (0.00 sec)

mysql> SELECT* FROM department;
+------+-----------+-----------------+-------------+
| d_id | d_name    | function        | address     |
+------+-----------+-----------------+-------------+
| 1001 | 科研部    | 新产品研发      | 3号楼5层    |
| 1002 | 生产部    | 主管生产        | 5号楼1层    |

```
两个表的d_id字段都是表示部门号,通过d_id字段对两表建立内连接查询
```
mysql> SELECT num,name,emploee.d_id,age,sex,d_name,function
    -> FROM emploee,department
    -> WHERE emploee.d_id=department.d_id;
```

##外连接查询
分为左连接查询，右连接查询
###左连接查询
```
mysql> SELECT num,name,emploee.d_id,age,sex,d_name,function FROM emploee LEFT JOIN department ON emploee.d_id=department.d_id;
+------+--------+------+------+------+-----------+-----------------+
| num  | name   | d_id | age  | sex  | d_name    | function        |
+------+--------+------+------+------+-----------+-----------------+
|    1 | 张三   | 1001 |   26 | 男   | 科研部    | 新产品研发      |
|    2 | 李思   | 1001 |   24 | 女   | 科研部    | 新产品研发      |
|    1 | 天天   | 1001 |   22 | 男   | 科研部    | 新产品研发      |
|    3 | 王五   | 1002 |   25 | 男   | 生产部    | 主管生产        |
|    4 | Aric   | 1003 |   15 | 男   | NULL      | NULL            |
+------+--------+------+------+------+-----------+-----------------+
5 rows in set (0.00 sec)
```
可以查询出emploee中所有记录，而department中只能查询出匹配的记录
###右连接查询
和左连接查询类似。可以查询出department中所有记录，而emploee中只能查询出匹配的记录
##复合条件连接查询
增加多个条件，使查询结果更加精确
```
mysql> SELECT num,name,emploee.d_id,age,sex,d_name,function FROM emploee,department WHERE emploee.d_id=department.d_id AND age > 24;
+------+--------+------+------+------+-----------+-----------------+
| num  | name   | d_id | age  | sex  | d_name    | function        |
+------+--------+------+------+------+-----------+-----------------+
|    1 | 张三   | 1001 |   26 | 男   | 科研部    | 新产品研发      |
|    3 | 王五   | 1002 |   25 | 男   | 生产部    | 主管生产        |
+------+--------+------+------+------+-----------+-----------------+
2 rows in set (0.00 sec)
```
还可以使用ORDER BY,GROUP BY
